# Worked Examples Library — Canonical Case Studies

**Purpose**: Reference library of 12 canonical worked-case studies
spanning the full sensitivity spectrum (Tier 1 / 2 / 3, single-topic
to multi-axis compound). Each example demonstrates the OUTPUT STYLE
the AI should emulate.

**Source**: Synthesis of Batches 1-5 doctrine + real-world consultation
patterns + canonical KP teaching cases (KSK Reader case studies +
modern KP practitioner examples).

**Loaded universally** in ADVANCED_FILES (PR B6.2). These are
REFERENCE PATTERNS — the AI doesn't reproduce them verbatim but
models its output structure on them.

## INTRODUCTION

This file is the AI's "modeling library." For each common question
type, it shows:
- Question template
- Tier sensitivity
- Output structure
- Key doctrinal phrases
- Cross-links used
- Close-on-hope framing

The AI applies these PATTERNS to actual chart-specific reading, NOT
the example chart specifics.

## META: HOW TO USE THIS LIBRARY

The AI should:

1. RECOGNIZE the question type from these patterns
2. EMULATE the output structure (not copy verbatim — adapt to client's chart)
3. APPLY the relevant cross-links + KB depth
4. PRESERVE the framing voice (compassionate, structural, decision-
   supportive, never verdict)
5. CITE crisis resources where applicable
6. CLOSE on agency + hope

These examples illustrate the QUALITY BAR. Every chart is unique;
the output adapts to the specific chart while preserving the
structural quality these examples demonstrate.
